# План улучшений ФотоПочта

## Статус сборки
✅ `flutter build apk --debug` — УСПЕШНО. APK готов к установке.

---

## Задачи

### 1. Название приложения в APK
- **Файл:** `android/app/src/main/res/values/strings.xml`
- **Что:** Изменить `app_name` на `ФотоПочта`
- **Файл сборки:** `android/app/build.gradle` — добавить `applicationId` и версию

---

### 2. "Разработал Алексей Майоров"
- **Где:** Экран «О приложении» (новый) или в AppBar drawer/меню
- **Что:** Добавить новый раздел настроек «О приложении» с:
  - Название: ФотоПочта
  - Разработчик: Алексей Майоров
  - Версия приложения

---

### 3. Переработка главного экрана — структура и логика

**Текущий порядок секций (плохой):**
1. Параметры сообщения
2. Добавление фото
3. Выбранные фотографии
4. Отправка сообщения
5. Аналитика

**Новый порядок (логичный):**
1. **Добавить фото** ← источник добавления фото ЗДЕСЬ сразу
2. **Фотографии по письмам** ← ОБЪЕДИНЕНИЕ: миниатюры + к какому письму принадлежат + размер
3. **Письмо** ← параметры (кому, тема — переименовать «тему» и убрать непонятный «фотоотчёт»)
4. **Отправка** ← кнопки действий

---

### 4. Тема письма — «Фотоотчёт» непонятно
- **Проблема:** placeholder `Фотоотчет` и пункт «Добавить дату в тему» работают неочевидно
- **Решение:**
  - Переименовать label «Тема» → «Тема письма»
  - Убрать или переименовать подсказку `Фотоотчет` → показывать пример: `Фото 23.02.2025`
  - Сделать блок даты более понятным: «📅 Дата в теме: 23.02.2025 [изменить]»

---

### 5. Объединение «Выбранные фото» + «Аналитика»
**Новый виджет `_PhotosByBatchGrid`:**
- Группирует фото по письмам (батчам)
- Для каждого письма: заголовок «Письмо 1 — 3 фото — 5.1 МБ»
- Под заголовком: горизонтальная сетка миниатюр
- Цветовая полоса заполненности письма
- Возможность rotatePhoto прямо из сетки

---

### 6. Алгоритм сжатия
- **Текущее состояние:** есть поле `compressionPreset` но только значение `'none'`
- **Добавить варианты:**
  - `none` — Без сжатия (оригинал)
  - `light` — Лёгкое (80% качества JPEG)
  - `medium` — Среднее (60% качества)  
  - `strong` — Сильное (40%, для экономии трафика)
- **Файлы:** `settings_model.dart`, `settings_repository.dart`, `AttachmentPreparer.kt`

---

### 7. Улучшение валидации Яндекс-авторизации
**Текущая проблема:** Нет чёткой инструкции что именно создать
**Улучшения:**
- Пошаговая инструкция прямо в UI (3 шага)
- Кнопки «Открыть настройки Яндекс Почты» и «Создать пароль приложения»
- Валидация формата пароля (16 символов, допускаются дефисы)
- Показ сколько символов введено: `12/16`
- Предупреждение если пароль слишком короткий

---

### 8. Настройки — порядок пунктов в `_SendSettingsPage`
**Новый порядок:**
1. Источник добавления фото ← ПОДНЯТЬ ВВЕРХ
2. Лимит одного письма
3. Способ отправки
4. Почтовый клиент
5. Авторизация Яндекса (только если автоматический режим)
6. Автоматическая отправка
